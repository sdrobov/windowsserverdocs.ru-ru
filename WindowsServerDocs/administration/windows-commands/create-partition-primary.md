# <a name="create-partition-primary"></a>создать секцию PRIMARY

> Область применения: Windows Server (половина ежегодного канала), Windows Server 2019, Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Создает основной раздел на базовом диске с фокусом. После создания секции фокус автоматически перемещается на новую секцию.

> [!IMPORTANT]
> Для выполнения этой операции необходимо выбрать базовый диск. Чтобы выбрать базовый диск и переместить фокус на него, необходимо использовать команду [Выбор диска](select-disk.md) .

## <a name="syntax"></a>Синтаксис

```
create partition primary [size=<n>] [offset=<n>] [id={ <byte> | <guid> }] [align=<n>] [noerr]
```

### <a name="parameters"></a>Параметры

| Параметр | Описание |
| --------- | ----------- |
| Размер =`<n>` | Указывает размер раздела в мегабайтах (МБ). Если размер не указан, Секция будет продолжаться до тех пор, пока в текущем регионе не будет больше нераспределенного пространства. |
| offset =`<n>` | Смещение в килобайтах (КБ), в котором создается секция. Если смещение не задано, раздел будет начинаться в начале самого крупного экстента диска, достаточного для его хранения. |
| Выровняйте =`<n>` | Выравнивает все экстенты секций до ближайшей границы выравнивания. Обычно используется с массивами LUN аппаратного RAID-массива для повышения производительности. `<n>`количество килобайтов (КБ) от начала диска до ближайшей границы выравнивания. |
| Идентификатор = { `<byte>  | <guid>` } | Указывает тип секции. Этот параметр предназначен для использования только изготовителем оборудования (OEM). С этим параметром можно указать любой байт типа секции или GUID. DiskPart не проверяет тип секции на допустимость, за исключением того, что он является байтом в шестнадцатеричной форме или GUID. **Внимание!** Создание секций с этим параметром может привести к сбою или невозможности запуска компьютера. Если вы не являетесь поставщиком вычислительной техники или ИТ-специалистом, имеющим дело с дисками GPT, не создавайте разделы на GPT-дисках с помощью этого параметра. Вместо этого всегда используйте команду " [создать секцию EFI](create-partition-efi.md) " для создания СИСТЕМНЫХ разделов EFI, команду " [создать секцию MSR](create-partition-msr.md) " для создания зарезервированных секций Майкрософт и команду " [создать первичную секцию](create-partition-primary.md)" (без `id={ <byte>  | <guid>` параметра) для создания основных разделов на дисках GPT.<p>**Для дисков основной загрузочной записи (MBR)** необходимо указать байт типа раздела в шестнадцатеричной форме для секции. Если этот параметр не указан, команда создает секцию типа `0x06`, которая указывает, что файловая система не установлена. Примеры приведены ниже.<ul><li>**Раздел данных LDM:** 0x42</li><li>**Раздел восстановления:** 0x27</li><li>**Распознанная Секция OEM:** 0x12, 0X84, 0XDE, 0XFE, 0x82</li></ul><p>**Для дисков с таблицей разделов GUID (GPT)** можно указать GUID типа раздела для создаваемой секции. Распознаваемые идентификаторы GUID включают:<ul><li>**Системный раздел EFI:** c12a7328-f81f-11d2-ba4b-00a0c93ec93b</li><li>**Зарезервированный раздел Майкрософт:** e3c9e316-0b5c-4db8-817d-f92df00215ae</li><li>**Базовый раздел данных:** ebd0a0a2-b9e5-4433-87c0-68b6b72699c7</li><li>**Раздел метаданных LDM (динамический диск):** 5808c8aa-7e8f-42e0-85d2-e1e90434cfb3</li><li>**Раздел данных LDM (динамический диск):** af9b60a0-1431-4f62-bc68-3311714a69ad</li><li>**Раздел восстановления:** de94bba4-06d1-4d40-A16A-bfd50179d6ac<p>Если этот параметр не указан для диска GPT, команда создает базовый раздел данных.</li></ul> |
| Noerr | Только для сценариев. При возникновении ошибки DiskPart продолжит обрабатывать команды, как если бы ошибка не возникала. Без параметра noerr ошибка приводит к выходу из программы DiskPart с кодом ошибки. |

## <a name="examples"></a>Примеры

Чтобы создать основной раздел размером 1000 МБ, введите:

```
create partition primary size=1000
```
